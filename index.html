How it works

Four components, one autonomous pipeline

Profile goes in. Prioritised decision + personalised message comes out.

Step 01
📊
Lead scorer
Random Forest reads 12 LinkedIn signals. Outputs Hot/Warm/Cold + priority score 0-100.
scikit-learn
Step 02
📈
Reply predictor
TF-IDF + GBM predicts reply probability for any message + profile combination.
NLP + GBM
Step 03
🤖
Agent brain
Combines both scores. Autonomously decides: SEND / REVIEW / NURTURE / SKIP with full reasoning.
Agentic AI
Step 04
✍️
LLM writer
LLaMA 3 reads the profile and writes a personalised message with specific hooks from their bio.
LLaMA 3 · Groq
Integration

Plug directly into SBL's pipeline

One API call. Profile in. Score + decision + message out. No UI changes needed.

Option 1 — REST API
Call one endpoint
POST a LinkedIn profile to /agent. Get back the score, action, and personalised message. Integrate with SBL's existing campaign builder in one afternoon.
# Before launching a campaign POST /agent { "profile": { "job_title": "Founder", "industry": "SaaS", "bio": "Scaling B2B outbound..." } } # Response { "action": "SEND", "score": 87, "reply_prob": 88.4, "message": "Your SaaS scaling..." }
Option 2 — Batch API
Score entire lead lists
Send up to 50 leads in one request. Get back a sorted campaign report — SEND at top, SKIP at bottom. Pre-filter before any campaign launches.
# Batch score before campaign POST /agent/batch { "profiles": [ { "name": "Priya", ... }, { "name": "Rahul", ... }, // up to 50 profiles ] } # Response — sorted by priority { "summary": { "send": 12, "review": 8, "skip": 30, "efficiency": "40%" } }
Option 3 — Python SDK
Import directly
Clone the repo and import the pipeline module. Run locally or in any Python environment. No server needed for internal use.
# Direct Python import from src.agent_pipeline import run_campaign results = run_campaign(lead_profiles) # Returns sorted list # SEND leads at top # Each with message ready
Option 4 — Swagger UI
Test without code
Run the API locally and open /docs. Full interactive Swagger UI — test every endpoint, see request/response shapes, try live calls. No code needed to explore.
# Start the API uvicorn api:app --reload # Open in browser http://localhost:8000/docs # 5 endpoints available: GET /health POST /score POST /reply POST /personalise POST /agent POST /agent/batch
